What I've actually done

Two roles, two different kinds of engineering problems. Both involved owning things end to end.

Alt DRX
Software Engineer
Feb 2024 — Present
● Current
// The situation

The platform ran on Laravel and was showing its limits. Response times were climbing as load grew, and the ORM layer wasn't helping. A migration to Node.js had been talked about for a while — I was brought in to lead it.

// What I did
  • Rebuilt the backend in Node.js and redesigned the data layer using Sequelize ORM — not just a port, but a proper re-think of how queries were structured
  • Ran both systems in parallel during the transition and did a clean cutover with zero downtime and no user-facing incidents
  • Built RESTful APIs and admin panel views in Vue.js for the ops team and founders
  • Worked on a biweekly release cycle in a cross-functional team — engineering, product, and operations
